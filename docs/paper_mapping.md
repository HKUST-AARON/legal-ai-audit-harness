The harness evaluates procedural qualification. It decides the highest status an output may claim from its visible legal-material chain, authority packet, contestability route and adoption record.

The full validation suite is the repository-level replication entry point. It reruns stress scenarios, public metadata records, public legal-system outputs, issue-specific public output/source packets, the endpoint-matched public retrieval benchmark, out-of-sample holdout packets, raw Codex model outputs, raw-output transcript anchors, cross-engine raw model outputs, cross-engine transcript anchors, source-supported model-output repairs, cross-engine source-supported repairs, the model-output evidence ladder, adversarial source-support repairs, public source-text anchor checks, formal invariant verification, status-lattice exhaustion, metric separation analysis with bootstrap/permutation robustness, baseline rule comparison, qualified-output gate ablations, gate-contrast witness pairs, qualified-output source-chain attacks, dynamic contestation challenges, policy-constants replay, metamorphic policy tests, model-identity invariance, query-perturbation stability diagnostics, query-portfolio frontier analysis, blocked-claim repair frontiers, jurisdiction-profile mutations, ranking-visibility diagnostics, proof-carrying status certificates, certificate tamper-resistance, mixed-authority source-screening packets, issue ablations, annotation robustness recoding, annotation-uncertainty Monte Carlo perturbation, score-blinded coding and full-threshold sensitivity, then writes one aggregate validation report. It is scenario-regression and artifact validation: the base `scores` vector is human-coded, while the harness computes status allocation, role caps, set-overlap metrics, evidence-packet metrics, source-tag checks, procedural-source binding and failure caps. The cross-engine layer runs the same three issue families through three model engines: all nine high-recall raw outputs are capped at reference information, all nine source-supported repairs qualify as normative material screening, and all thirty-six transcript locators are anchored in committed raw transcripts. The holdout layer contains eight withheld public-retrieval packets, eight raw model-output packets and eight source-bound repair packets generated after the scoring policy is frozen; expected statuses are not stored in those scenario files. The formal invariant layer exhaustively checks that higher procedural status cannot be obtained without authority-set, evidence-packet, counter-material, review-gate, role-cap and adoption-gate conditions. The status-lattice layer exhausts 233,280 high-status claim-attempt states, 1,632,960 cover edges and 1,632,960 substitute-rule predictions; it proves 851/851 high-status necessity checks and 851/851 gate-ablation drops, while every simplified score, role, source or authority substitute over-admits unless the full screening predicate is present. The metric-separation layer computes threshold confusion matrices showing that upstream precision, recall and F1 alone are weak predictors of procedural qualification, then bootstraps the key estimates and permutes recall-status association. The baseline-comparison layer applies 12 alternative status rules over 2,988 predictions and shows that recall, F1, total-score, source-bound and review-gate substitutes all over-admit or miss procedurally qualified packets. The aggregate report now emits a substitute-theory falsification table for performance sufficiency, source-label sufficiency, review-label sufficiency, score sufficiency and model-identity sufficiency; all five over-admit outputs, while the full protocol records zero false positives. The gate-ablation layer removes evidence, source-tag, authority, counter-material, review, role-cap and adoption gates from all qualified packets and verifies that status falls below the corresponding procedural level. The gate-contrast witness layer proves Proposition 1 directly: it preserves audit score vectors, upstream metrics and system role while flipping one mandatory predicate, and every witness pair separates procedural status. The source-chain attack layer applies every nonempty combination of locator, output-source link, procedural source tag, high-authority recall and counter-material recall attacks across every qualified packet. The contestation challenge layer applies valid counter-material, source-verification, jurisdiction and contestability-channel challenges plus unsupported challenge controls across every qualified packet. The policy-constants replay layer recomputes status in a second Python script parameterized by `policy/legal_output_policy.json` thresholds, role caps, protected reliance gates, source-tag classes, jurisdiction aliases and failure dispositions, without importing the harness model, then compares 4,752 score, role-cap, missing-gate, failure-cap, metric and final-status fields against the emitted certificates. The certificate tamper-resistance layer mutates certificate identities, hashes, policy bodies, scores, roles, gates, status fields, caps, metrics, proof obligations and certificate-set structure; all 5,811 tamper cases are rejected. The metamorphic policy layer applies claim escalation, upstream-metric inflation, role-cap demotion, source-tag mutation, review-gate removal, score-and-role inflation without adoption, and benign-source augmentation without using expected labels. The model-identity invariance layer substitutes five model identity profiles across all 264 packets and records 1,320/1,320 unchanged status/disposition allocations; the substitute-theory table separately shows that model-identity sufficiency would over-admit 1,005 identity-labelled rows, proving model labels do not enter the status function. The query-perturbation layer compares 30 issue-equivalent public-search query variants across five issue groups to test whether status, authority coverage, top-result identity and retrieved-record sets remain stable under query reformulation. The query-portfolio layer enumerates 315 non-empty public-retrieval query portfolios plus five group frontier summaries and tests whether frozen query expansion can recover high-authority and counter-material coverage. The repair-frontier layer computes the minimal artefact-gate families needed to upgrade each blocked normative-screening or decision-support claim. The jurisdiction-profile layer checks that high-status claims declare profile-supported jurisdiction assumptions and that qualified packets downgrade when profile assumptions are absent or mismatched. The ranking-visibility layer computes a rank-window visibility curve for counter-material salience across high-status claims and applies rank-order counterfactuals where enough non-counter material exists to move counter-material below the visibility window. The status-certificate layer emits proof-carrying derivation objects with scenario identity, scenario hash, policy hash, policy body, jurisdiction profile, score gate, role cap, missing gates, failure cap, metric bundle, claim support, 17 proof obligations and a derivation hash for every packet. The claim-consistency verifier is an artifact integrity check: it compares headline counts in the manuscript, README, artifact guide, skill and generated reports against `full_validation_report.json`. The source-text anchor layer checks manifest support terms against extracted public source snapshots; the transcript-anchor layer checks raw model-output scenario locators against the committed model transcript and the cross-engine model transcript. The annotation robustness layer re-scores all 264 committed scenario packets under strict and lenient coding policies. The annotation-uncertainty layer perturbs all six audit scores 66,000 times under a fixed-seed noise model, reporting exact status stability, high-status boundary stability and fragile boundary cases. The score-blinded coding layer strips original scores and expected outcomes from 240 non-holdout packets, then compares status, inter-coder dimension-level, post-evaluation failure-flag and missing-gate agreement, and base-dimension calibration with Cohen's kappa, three-category PABAK and mean score drift across two codebook-based audit passes.

The implementation adopts the legal-workflow logic in `anthropics/claude-for-legal`: outputs are treated as draft or screening artefacts until review is complete; citations and source links are tagged by verification posture; jurisdiction assumptions are surfaced; and external reliance, filing, sending or execution requires accountable human authorization. The audit layer is runtime-agnostic and attaches to agents, RAG stacks, search systems, public databases and manual review workflows.

The public system output pilot uses ordered top-k records from committed public legal system snapshots. It validates that real upstream legal outputs can be frozen, source-tagged, reconstructed and status-qualified by the same certificate-producing audit layer.

The issue-specific public output/source study is stricter: it freezes two naturally returned public issue-search outputs and one source-bound public-source packet, then measures whether the visible records contain the relevant high-authority and counter-material materials. It is the bridge between source reconstruction and the curated issue packets.

The public retrieval benchmark is the non-tautological public-search layer: it runs twenty-two committed public query outputs from CourtListener, the National Archives, Supreme Court of Canada Lexum search, OpenLegalData and CURIA against endpoint-compatible high-authority and counter-material sets, then lets the harness compute coverage failures and procedural status. Eight additional public-retrieval outputs are withheld for the holdout layer. The query-perturbation layer combines those thirty public-retrieval variants by issue family and reports that all five issue groups preserve status, while three have unstable authority coverage, four have unstable returned-record sets and four have unstable top results. The query-portfolio layer then enumerates all 315 non-empty query portfolios plus five group frontier summaries: 56 recover full high-authority coverage, but none recover full counter-material coverage or qualify as screening. Mixed statute/case authority screening is tested by the source-screening packets instead of requiring case-only endpoints to return legislation.

The Codex output pilot evaluates ten `gpt-5.5` / `xhigh` legal authority recommendations. It shows that high authority coverage and counter-material coverage do not by themselves qualify raw model text as `normative_material_screening_output`: without source-support evidence, the harness caps the outputs at `reference_information`. The repair intervention keeps the same issue-defined authority set, validates each repaired output-source link against an issue manifest, locator, hashed manifest source-support excerpt, contradiction patterns, procedural source tag and issue-set coverage, and applies a separate rank-salience repair where an already retrieved limiting material sits outside the review window. Those variants qualify as `normative_material_screening_output`, which tests the paper's claim that source support and rank salience, not model identity, control procedural status. The adversarial repair layer then generates sixty negative controls over the same outputs and confirms that locator, source, claim, output-link and counter-material failures are rejected rather than upgraded.
